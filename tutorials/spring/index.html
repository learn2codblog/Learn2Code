Spring Bean

Written by Pradeep Kumar

A Spring Bean is an object that is managed by the Spring IoC (Inversion of Control) container. Beans are the backbone of a Spring-based application; they are the objects that form the core of your application and are managed by the Spring framework. The Spring IoC container is responsible for instantiating, configuring, and assembling these beans.

Key Concepts of Spring Beans:

  • Definition: A bean is an object that is instantiated, assembled, and otherwise managed by a Spring IoC container. Beans are created from configuration metadata that you provide to the container, for example, in the form of XML configuration, Java annotations, or Java code.
  • Scope:
    • Singleton (default): A single instance of the bean is created and shared throughout the Spring container.
    • Prototype: A new instance is created every time the bean is requested.
    • Other scopes: request, session, application, and websocket, primarily used in web-aware applications.
  • Lifecycle: The lifecycle of a Spring bean is managed by the container. The typical lifecycle includes:
    • Instantiation: The bean is created.
    • Dependency Injection: Dependencies are injected into the bean.
    • Initialization: Custom initialization methods can be invoked.
    • Usage: The bean is ready to be used.
    • Destruction: The bean is destroyed when the container is closed.

Declaring a Spring Bean

Beans can be declared in various ways in Spring:

  1. Using XML Configuration: 
    <bean id="myBean" class="com.example.MyBeanClass"/>
  2. Using Java Annotations:
    • @Component: Generic stereotype for any Spring-managed component.
    • @Service: Indicates that an annotated class is a "Service" (business service facade or service layer).
    • @Repository: Indicates that an annotated class is a "Repository" (data access layer).
    • @Controller: Indicates that an annotated class is a "Controller" (presentation layer).
    • @Configuration: Indicates that a class declares one or more @Bean methods.
    • @Bean: Indicates that a method produces a bean to be managed by the Spring container.
    import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AppConfig {
    @Bean
    public MyBeanClass myBean() {
        return new MyBeanClass();
    }
}
  3. Using Java Configuration (Java-based Configuration):

    Spring provides a way to define beans using Java code, which is often considered type-safe and refactor-friendly.

    import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AppConfig {
    @Bean
    public MyBeanClass myBean() {
        return new MyBeanClass();
    }
}

    Example Bean with Annotations

    package com.example;

import org.springframework.stereotype.Component;

@Component
public class MyBeanClass {
    public void doSomething() {
        System.out.println("Doing something...");
    }
}

    MainApplication.java

    package com.example;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.ApplicationContext;

@SpringBootApplication
public class MainApplication {
    public static void main(String[] args) {
        ApplicationContext context = SpringApplication.run(MainApplication.class, args);
        MyBeanClass myBean = context.getBean(MyBeanClass.class);
        myBean.doSomething();
    }
}

    In this example:

    • @Component marks MyBeanClass as a Spring bean.
    • MainApplication retrieves the bean from the Spring context and uses it.

    Summary

    A Spring Bean is an essential building block in a Spring application, managed by the Spring IoC container. Beans can be declared using XML, Java annotations, or Java configuration, and they have a lifecycle managed by the Spring container.

What is RestController Annotation

The @RestController annotation in Spring is a specialized version of the @Controller annotation. It is used to create RESTful web services using Spring MVC. This annotation is shorthand for @Controller and @ResponseBody combined. When you annotate a class with @RestController, it indicates that the class is a web controller, and all the methods in the class will return data directly to the caller in a suitable format (like JSON or XML) rather than rendering a view.

Key Characteristics of @RestController

  • Combines @Controller and @ResponseBody:
    • @Controller indicates that the class serves web requests.
    • @ResponseBody indicates that the return value of a method should be written directly to the HTTP response body as JSON or XML.
  • RESTful Services: It is typically used to create RESTful web services where the returned data is serialized into JSON or XML.
